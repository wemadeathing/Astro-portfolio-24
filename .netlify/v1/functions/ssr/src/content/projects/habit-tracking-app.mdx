## Project Overview
I needed a better way to track my coding projects and build better habits, but existing time trackers didn't fit how I work on creative projects. I started building this in Bolt for rapid prototyping, then moved it into Cursor when I wanted more control. It's designed around how I actually work, not some theoretical productivity system, and I use it daily to track my own development work.

## Key Features

### Productivity Tools
A clean, intuitive app that adapts to how I actually work on projects:

<ul class="list-none space-y-3 [&>li]:before:content-['•'] [&>li]:before:text-purple-600 [&>li]:before:mr-2 [&>li]:flex">
  <li>Focus Sessions - Intuitive time tracking that doesn't force rigid time blocks but adapts to natural work flows</li>
  <li>Project Analytics - Visual insights into productivity patterns across different types of coding and design work</li>
  <li>Habit Tracking - Simple habit building tools that track consistency without being overwhelming</li>
  <li>Activity Overview - Daily and weekly views that show actual work patterns rather than idealized schedules</li>
  <li>Real-time Sync - All data syncs instantly across devices using Supabase real-time capabilities</li>
  <li>Clean Interface - Minimal design that stays out of the way during focused work sessions</li>
</ul>

## Key Learnings
<ul class="list-none space-y-3 [&>li]:before:content-['•'] [&>li]:before:text-purple-600 [&>li]:before:mr-2 [&>li]:flex">
  <li>Building tools for yourself ensures they solve real problems you actually face</li>
  <li>Rapid prototyping with Bolt then moving to Cursor is an effective development workflow</li>
  <li>Personal productivity apps work best when they adapt to natural work patterns</li>
  <li>Real-time sync is essential for tools you use across multiple devices</li>
  <li>Simple, clean interfaces are more effective than feature-heavy dashboards</li>
  <li>Using your own tools daily provides the best feedback for iteration</li>
</ul>
